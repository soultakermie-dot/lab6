1. Мета роботи

Метою роботи є вивчення принципів роботи з колекціями в Java, освоєння технологій роботи з HashMap для зберігання пар ключ-значення, реалізація класу перекладача з можливістю додавання слів до словника та перекладу фраз, створення консольного інтерфейсу для взаємодії з користувачем та обробки введення даних з клавіатури.

2. Теоретичні відомості

HashMap - це клас колекції в Java, який реалізує інтерфейс Map та зберігає дані у вигляді пар ключ-значення. HashMap використовує хеш-таблицю для зберігання елементів, що забезпечує швидкий доступ до даних за ключем. Ключі в HashMap повинні бути унікальними, а значення можуть повторюватися. Операції додавання, видалення та пошуку елементів в HashMap виконуються за константний час O(1) в середньому випадку.

Колекції в Java - це набір класів та інтерфейсів, які надають структури даних для зберігання та обробки груп об'єктів. Колекції дозволяють працювати з даними більш ефективно та зручно, ніж використання масивів. Основні інтерфейси колекцій включають List, Set та Map, кожен з яких має свої особливості та призначення.

Робота з рядками в Java - клас String та методи для роботи з рядками дозволяють виконувати різні операції над текстом, такі як розбиття рядка на частини, перетворення регістру, пошук підрядків та інші. Метод split() дозволяє розбити рядок на масив підрядків за вказаним роздільником, що є корисним при обробці фраз та речень.

Консольний ввід-вивід - клас Scanner дозволяє читати дані з консолі та обробляти різні типи введення. Scanner надає методи для читання рядків, чисел та інших типів даних, що робить його зручним інструментом для створення інтерактивних консольних додатків.

Обробка помилок - при роботі з введенням користувача необхідно обробляти можливі помилки, такі як некоректний формат даних або порожні рядки. Правильна обробка помилок забезпечує стабільність програми та зручність використання.

3. Завдання

Створити консольний додаток, який:

- описує клас, який є перекладачем з англійської мови та містить словник англійських та українських слів у вигляді колекції HashMap<String, String>, метод для додавання пар слів та метод для перекладу отриманої фрази

- створює екземпляр описаного класу, наповнює словник (також підтримує введення даних з клавіатури)

- вводить деяку фразу англійською мовою та відображає її переклад українською мовою

4. Модель програми

Програма складається з двох основних компонентів.

Перший компонент - клас Translator відповідає за логіку перекладу. Він містить поле dictionary типу HashMap<String, String>, де ключ - це англійське слово, а значення - це український переклад. Конструктор класу ініціалізує словник як новий екземпляр HashMap. Метод addWordPair(String englishWord, String ukrainianWord) додає пару слів до словника, перетворюючи англійське слово на нижній регістр для забезпечення нечутливості до регістру. Метод translate(String phrase) приймає фразу англійською мовою, розбиває її на окремі слова за пробілами, для кожного слова перевіряє наявність його в словнику та замінює на переклад, якщо слово знайдено. Слова, яких немає в словнику, залишаються без змін. Метод також зберігає розділові знаки з оригінальної фрази.

Другий компонент - клас TranslatorApp реалізує консольний інтерфейс для взаємодії з користувачем. Він містить статичні поля scanner для читання введення з клавіатури та translator для роботи з перекладачем. Метод main() є точкою входу програми та містить головний цикл меню, який дозволяє користувачу вибирати дії: додавання пар слів до словника, переклад фрази або вихід з програми. Метод fillDefaultDictionary() наповнює словник базовими словами для демонстрації роботи перекладача. Метод addWordPair() дозволяє користувачу вводити пари слів з клавіатури та додавати їх до словника з перевіркою на порожні рядки. Метод translatePhrase() дозволяє користувачу вводити фразу англійською мовою та отримувати її переклад українською мовою. Метод getIntInput() забезпечує безпечне введення цілих чисел з обробкою помилок.

5. Хід виконання роботи

Спочатку я створив клас Translator, який містить словник типу HashMap<String, String> для зберігання пар англійських та українських слів. Конструктор класу ініціалізує словник як новий екземпляр HashMap. Метод addWordPair() приймає два параметри - англійське слово та український переклад, перевіряє їх на null та додає пару до словника, перетворюючи англійське слово на нижній регістр для забезпечення нечутливості до регістру при пошуку.

Метод translate() реалізує логіку перекладу фрази. Він перевіряє, чи фраза не є null або порожньою, потім розбиває фразу на окремі слова за допомогою методу split() з регулярним виразом для розбиття за пробілами. Для кожного слова метод видаляє розділові знаки для пошуку в словнику, перевіряє наявність слова в словнику та замінює його на переклад, якщо слово знайдено. Розділові знаки зберігаються та додаються до перекладеного слова. Слова, яких немає в словнику, залишаються без змін. Результат формується у вигляді рядка з перекладеними словами.

Далі створив клас TranslatorApp, який реалізує консольний інтерфейс. Клас містить статичні поля scanner для читання введення та translator для роботи з перекладачем. Метод main() містить головний цикл меню, який відображає користувачу список доступних дій та обробляє вибір користувача. Метод fillDefaultDictionary() наповнює словник базовими словами, такими як hello, world, good, morning та інші, для демонстрації роботи перекладача.

Метод addWordPair() дозволяє користувачу вводити пари слів з клавіатури. Він запитує англійське слово та український переклад, перевіряє їх на порожні рядки та додає пару до словника, виводячи підтвердження додавання. Метод translatePhrase() дозволяє користувачу вводити фразу англійською мовою, викликає метод translate() перекладача та виводить результат перекладу. Метод getIntInput() забезпечує безпечне введення цілих чисел з обробкою винятків NumberFormatException та повторним запитом введення при некоректних даних.

6. Результати виконання

Програма успішно виконує всі необхідні операції:

1. Створює клас Translator з словником типу HashMap<String, String> для зберігання пар англійських та українських слів

2. Реалізує метод addWordPair() для додавання пар слів до словника з перевіркою на null та перетворенням на нижній регістр

3. Реалізує метод translate() для перекладу фраз, який розбиває фразу на слова, шукає кожне слово в словнику та замінює його на переклад, зберігаючи розділові знаки

4. Створює консольний інтерфейс з меню, який дозволяє користувачу вибирати дії: додавання пар слів, переклад фрази або вихід

5. Наповнює словник базовими словами для демонстрації роботи перекладача

6. Підтримує введення даних з клавіатури для додавання нових пар слів до словника

7. Дозволяє вводити фрази англійською мовою та відображає їх переклад українською мовою

8. Обробляє помилки введення та забезпечує коректну роботу програми при некоректних даних

Всі операції виконуються коректно, словник працює правильно, переклад фраз виконується з урахуванням слів, які є в словнику. Консольний інтерфейс забезпечує зручну взаємодію з користувачем та дозволяє динамічно розширювати словник новими парами слів.

7. Висновки

В ході виконання роботи було створено консольний додаток на Java, який демонструє роботу з колекціями HashMap та реалізує функціонал перекладача з англійської на українську мову. Програма використовує стандартні Java API та виконує всі необхідні операції.

Було освоєно:

- Роботу з колекціями: використання HashMap для зберігання пар ключ-значення, розуміння принципів роботи з хеш-таблицями, забезпечення швидкого доступу до даних за ключем

- Роботу з рядками: використання методів String для обробки тексту, розбиття рядків на частини за допомогою split(), перетворення регістру символів, збереження розділових знаків при обробці

- Консольний ввід-вивід: використання класу Scanner для читання даних з клавіатури, обробка різних типів введення, забезпечення зручної взаємодії з користувачем

- Обробку помилок: перевірка введених даних на коректність, обробка винятків при некоректному введенні, забезпечення стабільності програми

- Структурування коду: розділення логіки перекладу та інтерфейсу користувача на окремі класи, забезпечення модульності та підтримуваності коду

- Алгоритми обробки тексту: розбиття фрази на слова, пошук слів у словнику, заміна слів на переклади, збереження структури та розділових знаків оригінальної фрази

Завдання виконано успішно, програма працює коректно та демонструє усі необхідні функції. Реалізований перекладач забезпечує зручну роботу зі словником та переклад фраз, а консольний інтерфейс дозволяє користувачу легко взаємодіяти з програмою та розширювати словник новими парами слів.
